# hyby-tech-web

本技能用于把“技术文档/合同/交付说明/会议纪要”等材料，整理为**对外可展示**的项目案例页，并输出一个**零依赖、可离线打开、可持续追加**的静态站点目录。

## 交付约束（硬性）

- **纯静态**：只输出 `index.html + assets/`（允许图片等静态资源）
- **不使用** React / Vue / Svelte 等框架
- **不依赖** npm、node、打包、构建命令
- **不依赖外部 CDN**：字体/CSS/JS 均本地化；默认 0 JS
- **离线可用**：双击 `index.html` 可正常展示（所有资源用相对路径）
- **便于继续追加**：内容块按 section/card 可复制结构组织

## 推荐输出目录结构

输出项目目录建议命名：`case-YYYYMM-项目简称/`。

```
case-YYYYMM-项目简称/
  index.html
  assets/
    style.css
    images/
      hero.jpg
      ...
  docs/                 # 可选：脱敏后的源材料/记录
    source_redacted.md
    notes.md
```

## 内置模板

本技能内置静态模板，复制后直接替换文本即可：

- `assets/template/index.html`
- `assets/template/assets/style.css`
- `assets/template/assets/images/hero.jpg`（占位图，可替换）

模板视觉风格对齐参考页面（渐变背景、玻璃拟态卡片、胶囊按钮、响应式网格）。

## 内容脱敏与对外化规则

### 脱敏默认规则
- 客户/甲方：统一替换为“某国内有色金属某分厂”（或更泛化称谓）
- 人名、电话、邮箱、地址、合同号、报价/单价：删除或泛化
- 品牌/型号：可用“主流品牌/行业通用型号”替代（如需保留边界，改写为能力/等级/标准）

### 对外案例页结构（建议）
- **项目概述**：场景 + 一句话定位
- **建设目标**：统一接入/权限/联动/运维
- **方案要点**：平台能力 + 存储策略 + 可靠性
- **项目价值**：业务价值 + 技术价值
- **扩展机会**：二期/三期方向

## 执行流程（给 AnyGen 的步骤）

1. 读取用户提供材料，抽取关键信息
2. 按脱敏规则改写为对外表述
3. 复制 `assets/template/` 作为新项目目录
4. 将内容填充到 `index.html` 的各 section
5. 将图片放入 `assets/images/`，并在 hero 区引用
6. 本地打开自检：断点响应式、链接、相对路径
